FYP / Thesis Project

Making floorplan segmentation work on real photographs

A CubiCasa5K-based study on how phone-camera capture degrades floorplan segmentation, how photo-domain fine-tuning helps, and how structural metrics catch failures that pixel-level inspection can miss.

Problem Pipeline Results Figures Project brief Workflow notes

Problem

Floorplan segmentation models are commonly tested on clean raster floorplans. In practice, users may only have a photograph of a printed or displayed floorplan. That changes the input distribution: perspective distortion, shadows, blur, uneven lighting, cropping, and JPEG compression can all break the predicted wall structure.

This project treats the issue as a structural robustness problem. The question is not only whether the mask looks visually similar, but whether the walls remain connected enough to form enclosed, usable room regions.

Core research question

Can a CubiCasa-style floorplan segmentation model be adapted to photographed floorplans, and which photo-capture factors still cause structural failure?

Key Results

391 paired clean/photo floorplans evaluated
16.1% clean-valid plans became invalid after photo capture
77.8% of degraded cases triggered the wall-fragmentation rule
85.2% clean-input structural validity rate
78.3% photo-input structural validity rate
49 silent failures: photo invalid but still visually plausible by free-space ratio

Validity shift

Among 391 paired examples, 333 clean inputs passed the structural gate, compared with 306 photographed inputs. There were 63 clean-valid/photo-broken degradation cases and 36 photo recoveries.

Main failure mode

The dominant failure mode was wall fragmentation. In degraded cases, mean wall connected components increased from 13.2 on clean inputs to 33.5 on photographed inputs.

Visual Summary

Structural Gate

Metric Purpose
wall_cc Counts connected wall components. High values indicate wall fragmentation.
enclosure_count Checks whether the predicted walls create enclosed room-like spaces.
enclosed_free_ratio Measures how much free space lies inside enclosed regions.
outside_free_ratio Measures leakage: free space connected to the outside of the plan.
